Copy Constructor in C++

A copy constructor is a special constructor in C++ that initializes a new object using an existing object of the same class. It is used to create a copy of an object.

Syntax of Copy Constructor

cpp
class ClassName {
public:
    ClassName(const ClassName &obj) {
        // Copy constructor code
    }
};

Types of Copy Constructors

  1. Default Copy Constructor (Shallow Copy)

    • Provided by the compiler if no copy constructor is defined.
    • Performs a member-wise copy (shallow copy).
    • Issue: If the class contains dynamically allocated memory, multiple objects may share the same memory, causing unintended side effects.

  2. User-Defined Copy Constructor (Deep Copy)

    • Explicitly defined by the programmer.
    • Used to perform a deep copy (allocating new memory and copying data).
    • Prevents issues related to shallow copy.

Why Use Copy Constructor?

  • To initialize an object from another object of the same class.
  • To pass objects by value to functions.
  • To return objects by value from functions.
  • To avoid shallow copy issues when using dynamic memory.

Example: Default vs. User-Defined Copy Constructor

cpp
#include <iostream>
#include <cstring>

class Student {
    char *name;
public:
    // Constructor
    Student(const char *n) {
        name = new char[strlen(n) + 1];
        strcpy(name, n);
    }

    // Copy Constructor (Deep Copy)
    Student(const Student &s) {
        name = new char[strlen(s.name) + 1];
        strcpy(name, s.name);
    }

    // Display function
    void display() {
        std::cout << "Name: " << name << std::endl;
    }

    // Destructor
    ~Student() {
        delete[] name;
    }
};

int main() {
    Student s1("Surya");
    Student s2 = s1;  // Calls copy constructor

    s1.display();
    s2.display();

    return 0;
}

Key Takeaways

  • A copy constructor ensures proper copying of objects, especially when dynamic memory is involved.
  • It prevents issues like double deletion and dangling pointers that occur in shallow copies.
  • Deep copy helps maintain separate memory for each object, avoiding unintended changes.

Shallow Copy vs. Deep Copy in C++

When copying objects in C++, there are two ways to copy data: shallow copy and deep copy. These concepts are essential when dealing with dynamically allocated memory.

1. Shallow Copy

What is a Shallow Copy?

  • A shallow copy copies all member variables of an object as they are.
  • It simply copies the memory addresses of dynamically allocated members instead of duplicating the actual data.
  • This means that both the original and copied object share the same memory location for dynamically allocated members.

Example of Shallow Copy

The default copy constructor provided by the compiler performs a shallow copy.

cpp
#include <iostream>
#include <cstring>

class Student {
    char *name;  // Pointer to dynamically allocated memory
public:
    Student(const char *n) {  
        name = new char[strlen(n) + 1]; // Dynamic memory allocation
        strcpy(name, n);
    }

    // Default Copy Constructor (Shallow Copy)
    Student(const Student &s) {
        name = s.name;  // Copies the memory address, not the actual content
    }

    void display() {
        std::cout << "Name: " << name << std::endl;
    }

    ~Student() {
        delete[] name; // Freeing memory
    }
};

int main() {
    Student s1("Surya");
    Student s2 = s1;  // Shallow copy

    s1.display();
    s2.display();

    return 0;
}

Problem with Shallow Copy

  • Both s1 and s2 point to the same memory location.
  • When s2 goes out of scope, the destructor deletes the memory, but s1 still has a pointer to that memory (dangling pointer issue).
  • If s1 is later destroyed, it will try to delete the same memory again, leading to a double deletion error and undefined behavior.

2. Deep Copy

What is a Deep Copy?

  • A deep copy creates a completely independent copy of an object.
  • Instead of just copying the memory address, it allocates new memory and copies the data into it.
  • This ensures that the original and copied objects have separate memory locations.

Example of Deep Copy

To perform a deep copy, we need to define our own copy constructor.

cpp
#include <iostream>
#include <cstring>

class Student {
    char *name;
public:
    Student(const char *n) {  
        name = new char[strlen(n) + 1];
        strcpy(name, n);
    }

    // Deep Copy Constructor
    Student(const Student &s) {
        name = new char[strlen(s.name) + 1]; // Allocate new memory
        strcpy(name, s.name);  // Copy the actual data
    }

    void display() {
        std::cout << "Name: " << name << std::endl;
    }

    ~Student() {
        delete[] name; // Free memory
    }
};

int main() {
    Student s1("Surya");
    Student s2 = s1;  // Calls deep copy constructor

    s1.display();
    s2.display();

    return 0;
}

Advantages of Deep Copy

✅ Each object has its own memory for dynamically allocated members.
✅ No risk of dangling pointers or double deletion.
✅ Ensures data integrity, even when dealing with complex objects.

Why Do We Need Shallow Copy and Deep Copy?

FeatureShallow Copy (Default Copy Constructor)Deep Copy (User-Defined Copy Constructor)
Memory HandlingCopies memory address (pointer)Allocates new memory and copies data
OwnershipBoth objects share the same memoryEach object has its own memory
RiskDouble deletion, dangling pointersNo risk, safe handling of memory
PerformanceFaster (no extra allocation)Slower (requires memory allocation)
UsageSuitable when dynamic allocation is not usedNecessary when dynamic memory allocation is involved

Conclusion

  • If a class does not use dynamic memory allocation, a shallow copy is sufficient.
  • If a class uses dynamic memory allocation, a deep copy should be implemented to prevent memory issues.
  • The copy constructor plays a critical role in defining how objects are copied, and implementing a deep copy helps ensure proper memory management.

VTable and VPointer in C++ (Virtual Table and Virtual Pointer)

Introduction

In C++, VTable (Virtual Table) and VPointer (Virtual Pointer) are key components of runtime polymorphism, which is achieved through virtual functions in inheritance.

  • VTable (Virtual Table): A mechanism used to implement dynamic (runtime) polymorphism by maintaining function pointers for virtual functions.
  • VPointer (Virtual Pointer): A hidden pointer in a class that points to the VTable of that class.

Why Do We Need VTable and VPointer?

When we use virtual functions in a base class, C++ ensures that the correct function is called at runtime based on the object's actual type, not the pointer type. This is possible due to VTable and VPointer.

How VTable and VPointer Work?

  1. VTable (Virtual Table)

    • Each class with virtual functions has a VTable.
    • The VTable is an array of function pointers, where each pointer stores the address of a virtual function.
    • If a derived class overrides a virtual function, the corresponding entry in the VTable is updated to point to the derived class's function.

  2. VPointer (Virtual Pointer)

    • Each object of a class with virtual functions has a hidden VPointer (VPTR).
    • The VPTR is automatically inserted by the compiler into the object's memory.
    • The VPTR points to the VTable of the object's actual class.
    • At runtime, the function call is resolved using the VTable through the VPointer.

Example: VTable and VPointer in Action

cpp
#include <iostream>
using namespace std;

class Base {
public:
    virtual void show() {  // Virtual function
        cout << "Base class show function" << endl;
    }
};

class Derived : public Base {
public:
    void show() override {  // Overriding virtual function
        cout << "Derived class show function" << endl;
    }
};

int main() {
    Base *bptr;   // Pointer to Base
    Derived d;    // Object of Derived

    bptr = &d;    // Base class pointer points to Derived object
    bptr->show(); // Calls Derived class show() due to VTable

    return 0;
}

Output

pgsql
Derived class show function

Step-by-Step Execution:

  1. VTable Creation

    • The compiler creates a VTable for each class with virtual functions.
    • The VTable stores pointers to the virtual functions.

  2. VPointer Assignment

    • Each object of a class stores a hidden VPointer that points to the respective VTable.
    • In Base class: 
      rust
      VPTR -> [ &Base::show ]
    • In Derived class: 
      rust
      VPTR -> [ &Derived::show ]

  3. Runtime Function Resolution

    • bptr->show(); calls the function via VTable.
    • Since bptr is pointing to a Derived object, the Derived class VTable is used, calling Derived::show() instead of Base::show().

Memory Layout (Simplified Representation)

ObjectVPointer (VPTR)VTable Entry
Base ObjectPoints to Base VTable&Base::show
Derived ObjectPoints to Derived VTable&Derived::show

Key Properties of VTable and VPointer

✅ Only created when a class contains at least one virtual function.
✅ One VTable per class, but each object gets a separate VPointer.
✅ Allows dynamic dispatch (runtime polymorphism).
✅ If a function is not overridden in the derived class, the Base class function remains in the VTable.

Performance Impact of VTable and VPointer

AspectWithout Virtual FunctionsWith Virtual Functions
Function Call SpeedDirect call (fast)Indirect lookup via VTable (slower)
Memory UsageNo VTable, no VPTRExtra memory for VTable & VPTR
FlexibilityStatically boundDynamically bound at runtime

💡 Note: The overhead of VTable lookup is minimal but matters in performance-critical applications.

When to Use Virtual Functions (and VTable)?

  • When you need runtime polymorphism.
  • When you expect function calls to be resolved dynamically.
  • When designing base classes with derived class overrides.

When to Avoid Virtual Functions?

  • If polymorphism is not needed, avoid virtual functions to save memory.
  • If performance is critical, avoid the extra VTable lookup.

Conclusion

  • VTable is a lookup table storing virtual function pointers for a class.
  • VPointer is a hidden pointer in an object that points to its class's VTable.
  • Used for dynamic (runtime) polymorphism to ensure the correct function is called.
  • Introduces slight overhead but provides flexibility for object-oriented programming.

By understanding VTable and VPointer, you can write more efficient and well-structured polymorphic code in C++. 🚀

Operator Overloading in C++

1. What is Operator Overloading?

In C++, operator overloading allows us to redefine the behavior of operators (e.g., +, -, *, /, ==, etc.) for user-defined data types (classes/structs). This makes objects of custom classes work with operators just like primitive data types.

Example Without Operator Overloading

cpp
#include <iostream>
using namespace std;

class Complex {
public:
    int real, imag;
    
    Complex(int r, int i) : real(r), imag(i) {}
    
    void display() {
        cout << real << " + " << imag << "i" << endl;
    }
};

int main() {
    Complex c1(3, 2), c2(1, 7);
    Complex c3 = c1 + c2; // ERROR ❌ (Operator + is not defined for Complex)
    c3.display();
    return 0;
}

Why the Error?

  • The + operator works for built-in types (int + int, double + double).
  • But c1 + c2 gives an error because C++ does not know how to add Complex numbers.

2. Why Use Operator Overloading?

✔ Makes user-defined types behave like built-in types.
✔ Improves code readability and simplicity.
✔ Allows intuitive mathematical operations on objects.
✔ Reduces the need for function calls like add(c1, c2) and instead uses c1 + c2.

3. How to Overload an Operator?

To overload an operator:

  • Use the keyword operator followed by the symbol.
  • Define it as a member function or a friend function.

4. Operator Overloading Example

cpp
#include <iostream>
using namespace std;

class Complex {
public:
    int real, imag;

    Complex(int r = 0, int i = 0) : real(r), imag(i) {}

    // Overloading + operator
    Complex operator+(const Complex& obj) {
        return Complex(real + obj.real, imag + obj.imag);
    }

    void display() {
        cout << real << " + " << imag << "i" << endl;
    }
};

int main() {
    Complex c1(3, 2), c2(1, 7);
    Complex c3 = c1 + c2; // Now works correctly ✅
    c3.display(); // Output: 4 + 9i
    return 0;
}

How It Works?

  • operator+ is overloaded to add real and imaginary parts separately.
  • When c1 + c2 is executed, the overloaded operator+ function is called.

5. Types of Operator Overloading

(a) Member Function Overloading

✅ Used when the left operand is an object of the class.

cpp
class Sample {
public:
    int value;
    
    Sample(int v) : value(v) {}

    // Overloading + operator as a member function
    Sample operator+(const Sample& obj) {
        return Sample(value + obj.value);
    }
};

(b) Friend Function Overloading

✅ Used when the left operand is not an object of the class.

cpp
class Sample {
public:
    int value;

    Sample(int v) : value(v) {}

    // Declaring friend function
    friend Sample operator+(const Sample& obj1, const Sample& obj2);
};

// Friend function definition
Sample operator+(const Sample& obj1, const Sample& obj2) {
    return Sample(obj1.value + obj2.value);
}

6. Overloading Unary Operators (++, --, -, !)

Unary operators work on a single operand.

cpp
#include <iostream>
using namespace std;

class Count {
public:
    int value;

    Count(int v) : value(v) {}

    // Overloading ++ operator (prefix)
    void operator++() {
        ++value;
    }

    void display() {
        cout << "Value: " << value << endl;
    }
};

int main() {
    Count c(5);
    ++c;  // Calls operator++ function
    c.display(); // Output: Value: 6
    return 0;
}

7. Overloading Binary Operators (+, -, *, /, ==, !=)

These work on two operands.

cpp
class Sample {
public:
    int value;

    Sample(int v) : value(v) {}

    // Overloading * operator
    Sample operator*(const Sample& obj) {
        return Sample(value * obj.value);
    }
};

8. Overloading Relational Operators (==, !=, <, >)

cpp
#include <iostream>
using namespace std;

class Compare {
public:
    int value;

    Compare(int v) : value(v) {}

    // Overloading == operator
    bool operator==(const Compare& obj) {
        return value == obj.value;
    }
};

int main() {
    Compare a(10), b(10);
    
    if (a == b)
        cout << "Objects are equal" << endl;
    else
        cout << "Objects are not equal" << endl;

    return 0;
}

✔ Allows comparing objects directly.

9. Overloading Stream Operators (<<, >>)

These allow printing and input for custom objects.

cpp
#include <iostream>
using namespace std;

class Print {
public:
    int value;

    Print(int v) : value(v) {}

    // Overloading << operator for cout
    friend ostream& operator<<(ostream& out, const Print& obj);
};

// Definition
ostream& operator<<(ostream& out, const Print& obj) {
    out << "Value: " << obj.value;
    return out;
}

int main() {
    Print p(50);
    cout << p << endl; // Output: Value: 50
    return 0;
}

✔ cout << obj; prints obj.value.

10. Operators That Cannot Be Overloaded

🚫 :: (Scope resolution)
🚫 . (Member access)
🚫 .* (Pointer-to-member)
🚫 sizeof (Size of type)
🚫 typeid (Runtime type information)

11. When to Use Operator Overloading?

✅ When you want natural syntax for operations on objects.
✅ When using mathematical operations like +, -, *, etc.
✅ When implementing custom comparison (==, !=, <, >).
✅ When printing objects using << or reading input using >>.

When Not to Use Operator Overloading?

❌ If it makes the code complex or less readable.
❌ If overloading does not match the natural behavior of the operator.

12. Summary

✔ Operator overloading allows redefining operators for user-defined types.
✔ It improves readability and natural syntax for object operations.
✔ Supports unary, binary, and relational operators.
✔ Uses member functions or friend functions.
✔ Cannot overload ::, ., .*, sizeof, typeid.

🚀 With operator overloading, C++ objects behave more like built-in types! 🚀

1. Static Constructor in C++

What is a Static Constructor?

  • A static constructor initializes static data members of a class.
  • It is not supported directly in C++ like in C#.
  • Instead, we use a static function or a static member variable with a constructor.

Workaround for Static Constructor in C++

Since C++ does not support static constructors directly, we achieve similar functionality using a static member variable:

cpp
#include <iostream>
using namespace std;

class Demo {
private:
    static int staticVar;

public:
    // Static constructor alternative
    class StaticConstructor {
    public:
        StaticConstructor() {
            cout << "Static Constructor called! Initializing staticVar.\n";
            staticVar = 100;  // Initialize static members
        }
    };

    static StaticConstructor sc;  // Single instance to trigger constructor

    static void show() {
        cout << "Static Variable: " << staticVar << endl;
    }
};

// Definition of static member
int Demo::staticVar;
Demo::StaticConstructor Demo::sc;  // Triggers static constructor

int main() {
    Demo::show();  // Static Constructor is called before this
    return 0;
}

Key Points

✅ Runs only once, before any object is created.
✅ Ensures proper initialization of static members.
✅ Used as a workaround, since C++ does not have built-in static constructors.

2. Virtual Constructor in C++

What is a Virtual Constructor?

  • C++ does not support virtual constructors directly because:
    • A constructor is responsible for creating objects, and virtual functions require an already created object.
  • However, we can simulate a virtual constructor using a Factory method.

How to Simulate a Virtual Constructor?

Using a clone() function (Factory Pattern):

cpp
#include <iostream>
using namespace std;

class Base {
public:
    virtual Base* clone() = 0;  // Virtual constructor (Factory method)
    virtual void show() = 0;
    virtual ~Base() {}  // Virtual Destructor
};

class Derived : public Base {
public:
    Derived() { cout << "Derived Constructor\n"; }
    
    // Virtual constructor
    Base* clone() override {
        return new Derived(*this);  // Create a copy
    }

    void show() override {
        cout << "Derived class object\n";
    }
};

int main() {
    Base* obj1 = new Derived();
    Base* obj2 = obj1->clone();  // Virtual Constructor in action

    obj1->show();
    obj2->show();

    delete obj1;
    delete obj2;
    return 0;
}

Key Points

✅ Used for polymorphic object creation.
✅ Simulated using a virtual function (like clone()).
✅ Helpful in Factory design patterns.
✅ Avoids object slicing (when copying derived class objects as base class objects).

3. Static Destructor in C++

What is a Static Destructor?

  • C++ does not support static destructors, because:
    • Destructors are called when an object is destroyed.
    • Static members exist until the program ends.
  • However, we can use a static object with a destructor to achieve a similar effect.

Workaround for Static Destructor in C++

cpp
#include <iostream>
using namespace std;

class Demo {
public:
    Demo() { cout << "Constructor called\n"; }

    ~Demo() { cout << "Destructor called\n"; }

    static void staticFunction() {
        static Demo obj;  // Will be destroyed at program termination
    }
};

int main() {
    Demo::staticFunction();
    cout << "Main Function End\n";
    return 0; // obj is destroyed here (Static Destructor behavior)
}

Key Points

✅ Runs automatically when the program exits.
✅ Used to clean up static resources.
✅ Achieved using a static local object inside a function.

4. Virtual Destructor in C++

What is a Virtual Destructor?

  • If a class has virtual functions, its destructor must be virtual.
  • Ensures proper destruction of derived class objects when deleted using a base class pointer.

Example Without Virtual Destructor (Memory Leak)

cpp
#include <iostream>
using namespace std;

class Base {
public:
    Base() { cout << "Base Constructor\n"; }
    ~Base() { cout << "Base Destructor\n"; } // Not virtual

class Derived : public Base {
public:
    Derived() { cout << "Derived Constructor\n"; }
    ~Derived() { cout << "Derived Destructor\n"; } // Not called if Base is deleted
};

int main() {
    Base* obj = new Derived();
    delete obj;  // Only Base Destructor is called! Memory leak!
    return 0;
}

Corrected Version Using Virtual Destructor

cpp
class Base {

class Derived : public Base {
public:
    Derived() { cout << "Derived Constructor\n"; }
    ~Derived() { cout << "Derived Destructor\n"; }  // Now correctly called

    return 0;
}

Key Points

✅ Prevents memory leaks.
✅ Ensures both base and derived destructors are executed.
✅ Must be declared virtual in base class for proper deletion.

Compile-time vs. Run-time Polymorphism in C++

Polymorphism means "many forms." In C++, it allows a function or operator to behave differently based on the input type or object type.

1. Compile-time Polymorphism (Early Binding)

  • Also called static polymorphism.
  • The function to be executed is determined at compile time.
  • Achieved using:
    • Function Overloading
    • Operator Overloading
    • Templates

2. Run-time Polymorphism (Late Binding)

  • Also called dynamic polymorphism.
  • The function to be executed is determined at runtime.
  • Achieved using function overriding with virtual functions.

🚀 Why Do We Need Run-time Polymorphism if Compile-time Exists?

  1. Flexibility: Compile-time polymorphism requires fixed method calls at compile-time, but run-time polymorphism allows us to decide the function to be executed during execution.
  2. Extensibility: Allows derived classes to modify behavior without changing the base class.
  3. Code Reusability: A base class reference can hold objects of different derived classes, allowing generalized coding.
  4. Avoid Code Duplication: Instead of writing multiple functions for different types, we override methods dynamically.

📌 Compile-time Polymorphism Example (Function Overloading)

cpp
#include <iostream>
using namespace std;

class CompileTimePoly {
public:
    // Function Overloading: Same function name, different parameters
    void show(int a) {
        cout << "Integer: " << a << endl;
    }

    void show(double b) {
        cout << "Double: " << b << endl;
    }

    void show(string c) {
        cout << "String: " << c << endl;
    }
};

int main() {
    CompileTimePoly obj;
    obj.show(5);        // Calls show(int)
    obj.show(5.5);      // Calls show(double)
    obj.show("Hello");  // Calls show(string)
    return 0;
}

Flow Explanation

  • At Compile Time, the compiler determines which show() function to call based on the argument type.
  • No runtime decision-making is required.
  • Efficient and Fast.

📌 Run-time Polymorphism Example (Function Overriding with Virtual Functions)

cpp
#include <iostream>
using namespace std;

class Base {
public:
    virtual void show() {  // Virtual function (Allows runtime binding)
        cout << "Base class function" << endl;
    }
};

class Derived : public Base {
public:
    void show() override {  // Overrides base class function
        cout << "Derived class function" << endl;

};

    Base* basePtr;     // Pointer of base class
    Derived d;
    basePtr = &d;      // Base class pointer points to Derived object

  1. Base Class Pointer (basePtr) points to a Derived Class Object (d).
  2. basePtr->show(); is executed.
  3. Since show() is a virtual function, the compiler checks the object type at runtime.
  4. Instead of calling Base::show(), it calls Derived::show() dynamically.

📌 How Does Run-time Polymorphism Work? (VTable and VPointer Mechanism)

🔹 What is VTable (Virtual Table)?

  • A VTable (Virtual Table) is a table that stores pointers to virtual functions of a class.
  • Every class with at least one virtual function has a VTable.

🔹 What is VPointer (Virtual Pointer)?

  • Each object of a class with virtual functions has a hidden pointer (vptr) pointing to the VTable.
  • At runtime, vptr is used to fetch the correct function from the VTable.

🔹 VTable and VPointer Working Mechanism

  1. For a Base Class (with a virtual function), the compiler creates a VTable for it.
  2. When a Derived Class overrides the virtual function, its VTable updates the function pointer.
  3. When calling a virtual function via a Base class pointer, the vptr determines the actual function at runtime.

📌 VTable and VPointer Example

cpp
#include <iostream>

public:
    virtual void show() {  // Virtual function
        cout << "Base class function" << endl;
    }
};

class Derived : public Base {
public:
    void show() override {  // Overrides base function
        cout << "Derived class function" << endl;
    }
};

int main() {
    Base* bptr;
    Derived d;
    bptr = &d;

    bptr->show();  // Uses VTable and vptr to call Derived::show()
    return 0;
}

📌 Execution Flow

  1. Base Class Object Creation
    • Base has a VTable with show() pointing to Base::show().
  2. Derived Class Object Creation
    • Derived overrides show(), so its VTable updates the function pointer.
  3. Calling bptr->show()
    • bptr points to a Derived object.
    • The vptr fetches the correct function from Derived’s VTable.
    • Derived::show() is executed dynamically.

🚀 When to Use Which?

✅ Use Compile-time Polymorphism when:

  • Behavior is fixed.
  • Performance is a priority (as it avoids vptr overhead).

✅ Use Run-time Polymorphism when:

  • Behavior must change dynamically based on the actual object type.
  • You want to achieve polymorphism in OOP.

💡 Final Thought:
Run-time Polymorphism (Virtual Functions) is essential for Object-Oriented Programming, Dynamic Behavior, and Code Reusability. Without it, we'd have to manually check object types at runtime, making code inefficient and less maintainable.

Virtual and Pure Virtual Functions in C++ (Detailed Explanation)

1️⃣ What is a Virtual Function?

A virtual function is a member function in a base class that you can override in a derived class. It allows run-time polymorphism, meaning the function to be executed is determined at runtime, not at compile-time.

🔹 Key Properties:

  • Declared using the virtual keyword in the base class.
  • Allows function overriding in the derived class.
  • Supports dynamic dispatch (function binding at runtime).
  • Uses VTable (Virtual Table) and VPointer (Virtual Pointer) Mechanism.

📌 Example of Virtual Function

cpp
#include <iostream>
using namespace std;

class Base {
public:
    virtual void show() {  // Virtual function
        cout << "Base class function" << endl;
    }
};

class Derived : public Base {
public:
    void show() override {  // Overriding base function
        cout << "Derived class function" << endl;
    }
};

int main() {
    Base* bptr;  // Base class pointer
    Derived d;
    bptr = &d;   // Base pointer points to Derived object
    bptr->show();  // Calls Derived::show() (Runtime decision)
    return 0;
}

📌 Execution Flow (VTable and VPointer)

  1. VTable (Virtual Table) is created for both Base and Derived classes.
  2. VPointer (vptr) is added to each object to point to the respective VTable.
  3. When calling bptr->show();:
    • Without virtual, it would call Base::show() (Compile-time binding).
    • With virtual, it dynamically calls Derived::show() at runtime (Run-time polymorphism).

2️⃣ What is a Pure Virtual Function?

A pure virtual function is a virtual function that has no implementation in the base class. It forces derived classes to override it.

🔹 Key Properties:

  • Declared using = 0 in the base class.
  • Makes a class an abstract class (i.e., you cannot create an object of it).
  • Must be overridden in derived classes, or they also become abstract.

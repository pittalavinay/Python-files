1. Introduction to Object-Oriented Programming (OOP) in C++

What is Object-Oriented Programming (OOP)?

Object-Oriented Programming (OOP) is a programming paradigm that organizes software design around objects rather than functions or logic. Objects represent real-world entities and are instances of classes.

Key Features of OOP:

  1. Encapsulation Bundling data and functions together.
  2. Abstraction Hiding implementation details from the user.
  3. Inheritance Reusing existing code through class hierarchy.
  4. Polymorphism Allowing a single function or operator to behave differently in different contexts.

Why Use OOP?

  • Improves code reusability and modularity.
  • Enhances maintainability and scalability.
  • Mimics real-world scenarios, making code easier to understand.

Example:

Without OOP (Procedural Programming)

cpp
#include <iostream>
using namespace std;

string carBrand;
int carSpeed;

void displayCar() {
    cout << "Car Brand: " << carBrand << endl;
    cout << "Speed: " << carSpeed << " km/h" << endl;
}

int main() {
    carBrand = "Tesla";
    carSpeed = 120;
    displayCar();
    return 0;
}

Problems with this approach:

  • No data security (anyone can modify carSpeed or carBrand).
  • Code is not reusable (each car requires new variables).

With OOP (Using Classes and Objects)

cpp
#include <iostream>
using namespace std;

class Car {
public:
    string brand;
    int speed;

    void displayCar() {
        cout << "Car Brand: " << brand << endl;
        cout << "Speed: " << speed << " km/h" << endl;
    }
};

int main() {
    Car myCar;
    myCar.brand = "Tesla";
    myCar.speed = 120;
    myCar.displayCar();
    return 0;
}

Advantages of this approach:
Encapsulation (all data related to a car is inside the Car class).
Reusability (we can create multiple car objects easily).

Real-World Applications of OOP

  1. Banking System: Accounts, transactions, and customers are represented as objects.
  2. E-Commerce Websites: Users, products, and orders are objects.
  3. Game Development: Characters, weapons, and enemies are objects with their own behaviors.
  4. Social Media Apps: Users, posts, and messages are modeled as objects.

2. Classes and Objects in C++

What is a Class?

A class is a blueprint or template for creating objects. It defines the structure and behavior of objects.

  • It consists of data members (variables) and member functions (methods).
  • A class does not occupy memory until an object is created.

What is an Object?

An object is an instance of a class. It represents a real-world entity with specific properties (data members) and actions (member functions).

Defining a Class and Creating Objects

Syntax of a Class

cpp
class ClassName {
    // Access specifier
    public:  
        // Data members
        int data;
        
        // Member function
        void showData() {
            cout << "Data: " << data << endl;
        }
};

Creating an Object

cpp
ClassName obj;  // Object creation
obj.data = 10;  // Accessing data member
obj.showData(); // Calling member function

Example: Defining a Class and Creating an Object

cpp
#include <iostream>
using namespace std;

        cout << "Speed: " << speed << " km/h" << endl;
    }
};

int main() {
    Car myCar;  // Creating an object

    myCar.brand = "Tesla";
    myCar.speed = 120;

    myCar.displayCar();  // Calling function

    return 0;
}

yaml

Speed: 120 km/h

Access Specifiers in a Class

Access specifiers define the visibility of data members and member functions in a class.

Access SpecifierDescription
publicAccessible from anywhere.
privateAccessible only inside the class.
protectedAccessible in derived classes.

Example: Public and Private Access Modifiers

cpp
#include <iostream>
using namespace std;

class Car {
private:
    int speed;  // Private member

public:
    string brand;

    // Function to set speed
    void setSpeed(int s) {
        speed = s;
    }

    // Function to display details
    void displayCar() {
        cout << "Car Brand: " << brand << endl;
        cout << "Speed: " << speed << " km/h" << endl;
    }
};

int main() {
    Car myCar;
    myCar.brand = "BMW";
    
    // myCar.speed = 120; //  Error: speed is private
    myCar.setSpeed(120);  //  Correct way to access private member

    myCar.displayCar();

    return 0;
}

Output:

yaml
Car Brand: BMW  
Speed: 120 km/h

Note:

  • Private members cannot be accessed directly from outside the class.
  • They must be accessed using public methods (like setSpeed).

3. Encapsulation in C++

What is Encapsulation?

Encapsulation is the process of wrapping data and functions together into a single unit (class) while restricting direct access to some details of an object.

Key Points:
Data hiding is achieved using private access specifiers.
Access to private data is provided through getter and setter methods.
Prevents accidental modification of critical data.

Real-World Example of Encapsulation

Encapsulation is like a bank ATM:

  • The ATM hides internal details (PIN verification, balance deduction).
  • Users can only access the required functions (withdraw, deposit).
  • Prevents direct access to sensitive data (account balance).

Example 1: Encapsulation Using Private Data Members

cpp
#include <iostream>
using namespace std;

class BankAccount {
private:
    double balance;  // Private variable

public:
    // Constructor to initialize balance
    BankAccount(double initialBalance) {
        balance = initialBalance;
    }

    // Getter method to check balance
    double getBalance() {
        return balance;
    }

    // Setter method to update balance safely
    void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
            cout << "Deposited: " << amount << ". New Balance: " << balance << endl;
        } else {
            cout << "Invalid deposit amount!" << endl;
        }
    }

    void withdraw(double amount) {
        if (amount > balance) {
            cout << "Insufficient funds!" << endl;
        } else {
            balance -= amount;
            cout << "Withdrawn: " << amount << ". New Balance: " << balance << endl;
        }
    }
};

int main() {
    BankAccount account(5000.0);

    account.deposit(2000.0);   //  Allowed
    account.withdraw(3000.0);  //  Allowed

    // account.balance = 10000; //  Error: Cannot access private data directly

    cout << "Final Balance: " << account.getBalance() << endl;

    return 0;
}

Output:

yaml
Deposited: 2000. New Balance: 7000
Withdrawn: 3000. New Balance: 4000
Final Balance: 4000

Encapsulation ensures that balance cannot be modified directly but only through deposit() and withdraw() functions.

cpp

Advantages of Encapsulation

Data Security: Prevents unauthorized access to sensitive data.
Code Maintainability: Changes can be made easily without affecting other parts of the program.
Flexibility and Reusability: Encapsulated classes can be reused in different applications.

Real-World Applications of Encapsulation

  1. Banking Systems Protects customer details (account balance, transactions).
  2. Medical Records Ensures only authorized users can modify patient data.
  3. Mobile Apps Hides complex logic behind simple interfaces (e.g., WhatsApp encrypts messages).
  4. Car Speedometer The speed calculation is hidden; users only see the result.

Key Takeaways

Encapsulation hides data using private access modifiers.
Data can only be accessed using public getter and setter methods.
Encapsulation ensures data security and prevents accidental modifications.

4. Abstraction in C++

What is Abstraction?

Abstraction is the process of hiding the implementation details and exposing only the essential features to the user.

Key Points:

Users interact with what a function does, not how it works.
Achieved using classes, access specifiers (private, public), and abstract classes.
Improves code readability and security by preventing direct access to sensitive data.

Real-World Example of Abstraction

Think of a car:

  • A driver only uses the steering, accelerator, and brake.
  • They dont need to know how the engine works internally.
  • The complexity is hidden, allowing for an easy interface.

Example 1: Abstraction Using Access Specifiers

cpp
#include <iostream>
using namespace std;

class Car {
private:
    int speed;  // Private data (hidden from the user)

public:
    // Constructor to set speed
    Car(int s) {
        speed = s;
    }

    // Public method (interface)
    void accelerate() {
        speed += 10;
        cout << "Speed increased. Current Speed: " << speed << " km/h" << endl;
    }
};

int main() {
    Car myCar(50);  // Create a car with initial speed 50

    myCar.accelerate();  //  Allowed: User interacts with accelerate() method

    // myCar.speed = 100; //  Error: Cannot access private member directly

    return 0;
}

Output:

bash
Speed increased. Current Speed: 60 km/h

Abstraction hides the speed variable and provides an interface (accelerate()).

Example 2: Abstraction Using Abstract Class and Pure Virtual Functions

An abstract class contains at least one pure virtual function, forcing derived classes to implement it.

Syntax for Abstract Class

cpp
class AbstractClass {
public:
    virtual void functionName() = 0; // Pure virtual function
};

Example: Online Payment System

cpp
#include <iostream>
using namespace std;

// Abstract class
class Payment {
public:
    virtual void processPayment() = 0; // Pure virtual function (no implementation)
};

// Derived class implementing the abstract function
class CreditCardPayment : public Payment {
public:
    void processPayment() override {
        cout << "Processing credit card payment..." << endl;
    }
};

class UpiPayment : public Payment {
public:
    void processPayment() override {
        cout << "Processing UPI payment..." << endl;
    }
};

int main() {
    CreditCardPayment cc;
    UpiPayment upi;

    cc.processPayment();  // Calls derived class method
    upi.processPayment(); // Calls derived class method

    return 0;
}

Output:

nginx
Processing credit card payment...
Processing UPI payment...

The Payment class enforces a processPayment() method, but the implementation is hidden from the user.

Advantages of Abstraction

Hides complexity Users dont need to know internal workings.
Improves maintainability Changes in internal code dont affect users.
Enhances security Prevents direct access to sensitive data.
Promotes reusability Abstract classes can be extended easily.

Real-World Applications of Abstraction

  1. ATM Machines Users interact with a simple interface; backend logic is hidden.
  2. Web Browsers Users enter a URL, but the rendering logic is abstracted.
  3. Smartphones Users see icons; the internal OS logic remains hidden.
  4. Gaming Engines Players interact with controls; game physics run in the background.

Key Takeaways

Abstraction hides unnecessary details while exposing only the required functionalities.
It can be achieved using access specifiers (private, public) or abstract classes.
Abstract classes enforce implementation of important methods in derived classes.

5. Inheritance in C++

What is Inheritance?

Inheritance is a feature in C++ that allows a class (child class) to acquire properties and behaviors from another class (parent class).

Key Points:

Promotes code reusability Write code once and reuse it in multiple classes.
Establishes a parent-child relationship between classes.
Helps in creating hierarchical structures in programs.

Real-World Example of Inheritance

Consider a vehicle hierarchy:

  • Parent Class: Vehicle (common properties like speed, fuel).
  • Child Class: Car, Bike, Truck (inherit common properties but have unique features).

Types of Inheritance in C++

  1. Single Inheritance One child inherits from one parent.
  2. Multiple Inheritance One child inherits from multiple parents.
  3. Multilevel Inheritance A child inherits from another child class.
  4. Hierarchical Inheritance One parent class is inherited by multiple child classes.
  5. Hybrid Inheritance Combination of multiple types.

Example 1: Single Inheritance

One class derives from a single base class.

cpp
#include <iostream>
using namespace std;

// Parent Class
class Vehicle {
public:
    int speed = 60;

    void showSpeed() {
        cout << "Speed: " << speed << " km/h" << endl;
    }
};

// Child Class (inherits from Vehicle)
class Car : public Vehicle {
public:
    string brand = "Toyota";

    void showBrand() {
        cout << "Car Brand: " << brand << endl;
    }
};

int main() {
    Car myCar;

    myCar.showSpeed();  //  Inherited from Vehicle
    myCar.showBrand();  //  Defined in Car

    return 0;
}

Output:

yaml
Speed: 60 km/h
Car Brand: Toyota

The Car class inherits showSpeed() and speed from Vehicle.

Example 2: Multiple Inheritance

A class inherits from two or more parent classes.

cpp
#include <iostream>

// Parent Class 1

// Child Class

        cout << "This is a car." << endl;
    }

int main() {
    Car myCar;
    myCar.engineType();  //  Inherited from Engine
    myCar.wheelCount();  //  Inherited from Wheels

Advantages of Inheritance

Code Reusability Common properties can be reused in multiple classes.
Faster Development No need to rewrite code for similar functionalities.
Enhances Maintainability Changes in the base class reflect in derived classes.
Promotes Hierarchical Class Design Helps in modeling real-world relationships.

Real-World Applications of Inheritance

  1. Banking System Account (Base Class) SavingsAccount, CheckingAccount (Derived Classes).
  2. E-commerce Platform Product (Base Class) Electronics, Clothing (Derived Classes).
  3. Hospital Management Person (Base Class) Doctor, Patient, Nurse (Derived Classes).
  4. Operating System Device (Base Class) Laptop, Mobile, Tablet (Derived Classes).

Key Takeaways

Inheritance allows one class to acquire the properties and behaviors of another class.
It promotes code reusability and efficient program structuring.
C++ supports multiple types of inheritance: Single, Multiple, Multilevel, Hierarchical, and Hybrid.

6. Polymorphism in C++

What is Polymorphism?

Polymorphism means "many forms" and allows functions or objects to behave differently based on the context.

Key Points:

Enables code flexibility and reusability.
Allows a single function name to have multiple behaviors.
Achieved using Function Overloading, Operator Overloading, and Function Overriding (Runtime Polymorphism).

Real-World Example of Polymorphism

Think of a smartphone's call function:

  • Calling via SIM.
  • Calling via WhatsApp or Skype.
  • Calling via Wi-Fi or cellular data.
    All are calls, but the method variesthis is polymorphism in action.

Types of Polymorphism in C++

1. Compile-Time Polymorphism (Static Binding)

  • Function Overloading
  • Operator Overloading

2. Runtime Polymorphism (Dynamic Binding)

  • Function Overriding (Using Virtual Functions)

1. Function Overloading (Compile-Time Polymorphism)

Function overloading allows multiple functions with the same name but different parameters.

Example: Function Overloading for Addition

cpp
#include <iostream>
using namespace std;

class MathOperations {
public:
    // Function to add two integers
    int add(int a, int b) {
        return a + b;
    }

    // Function to add three integers
    int add(int a, int b, int c) {
        return a + b + c;
    }

    // Function to add two double values
    double add(double a, double b) {
        return a + b;
    }
};

int main() {
    MathOperations obj;

    cout << "Sum of 2 and 3: " << obj.add(2, 3) << endl;
    cout << "Sum of 2, 3, and 4: " << obj.add(2, 3, 4) << endl;
    cout << "Sum of 2.5 and 3.5: " << obj.add(2.5, 3.5) << endl;

    return 0;
}

Output:

yaml
Sum of 2 and 3: 5
Sum of 2, 3, and 4: 9
Sum of 2.5 and 3.5: 6

Same function name (add) performs different operations based on input parameters.

2. Operator Overloading (Compile-Time Polymorphism)

C++ allows operators (+, -, *, etc.) to be overloaded for user-defined types.

Example: Overloading + Operator for Complex Numbers

cpp
#include <iostream>
using namespace std;

class Complex {
public:
    int real, imag;

    Complex(int r, int i) {
        real = r;
        imag = i;
    }

    // Overload the + operator
    Complex operator+(Complex obj) {
        return Complex(real + obj.real, imag + obj.imag);
    }

    void display() {
        cout << real << " + " << imag << "i" << endl;
    }
};

int main() {
    Complex c1(3, 4), c2(1, 2);
    Complex c3 = c1 + c2;  // Using overloaded + operator

    c3.display();  // Output: 4 + 6i

    return 0;
}

Output:

go
4 + 6i

The + operator now works for Complex numbers, just like it does for integers.

3. Function Overriding (Runtime Polymorphism)

Function overriding allows a derived class to modify a function from the base class.

Example: Function Overriding with Virtual Functions

cpp
#include <iostream>
using namespace std;

class Animal {
public:
    virtual void sound() {  // Virtual function
        cout << "Animals make sound" << endl;
    }
};

class Dog : public Animal {
public:
    void sound() override {  // Overriding function
        cout << "Dog barks" << endl;
    }
};

class Cat : public Animal {
public:
    void sound() override {  // Overriding function
        cout << "Cat meows" << endl;

    return 0;
}

7. Pure Virtual Functions in C++

What is a Pure Virtual Function?

A Pure Virtual Function is a function declared in a base class but implemented in derived classes. It forces derived classes to provide their own implementation of the function.

Key Points:

Declared in a base class using = 0.
Makes the base class abstract (cannot create objects of it).
Derived classes must override the pure virtual function.

Real-World Example of Pure Virtual Functions

Imagine a payment system where different payment methods exist (Credit Card, UPI, PayPal). A common function processPayment() is needed, but the implementation differs for each payment type.

Conceptual Representation:

  • Base Class: PaymentMethod (Declares processPayment())
  • Derived Classes: CreditCard, UPI, PayPal (Implement processPayment())

Syntax of Pure Virtual Function

cpp
class Base {
public:
    virtual void functionName() = 0;  // Pure virtual function
};

Example: Pure Virtual Function Implementation

cpp
#include <iostream>
using namespace std;

// Abstract Base Class
class PaymentMethod {
public:
    virtual void processPayment() = 0; // Pure virtual function
};

// Derived Class 1
class CreditCard : public PaymentMethod {
public:
    void processPayment() override {
        cout << "Processing payment through Credit Card." << endl;
    }
};

// Derived Class 2
class UPI : public PaymentMethod {
public:
    void processPayment() override {
        cout << "Processing payment through UPI." << endl;
    }
};

// Derived Class 3
class PayPal : public PaymentMethod {
public:
    void processPayment() override {
        cout << "Processing payment through PayPal." << endl;
    }
};

int main() {
    PaymentMethod* payment;

    CreditCard cc;
    UPI upi;
    PayPal paypal;

    // Processing payment using CreditCard
    payment = &cc;
    payment->processPayment();

    // Processing payment using UPI
    payment = &upi;
    payment->processPayment();

    // Processing payment using PayPal
    payment = &paypal;
    payment->processPayment();

    return 0;
}

Output:

nginx
Processing payment through Credit Card.
Processing payment through UPI.
Processing payment through PayPal.

Each derived class provides its own implementation of processPayment().

Abstract Classes in C++

A class that contains at least one pure virtual function is called an Abstract Class.

  • Cannot create objects of an abstract class.
  • Can use pointers and references to refer to derived class objects.

Example of Abstract Class

cpp
#include <iostream>
using namespace std;

class Animal {
public:
    virtual void sound() = 0; // Pure virtual function
};

class Dog : public Animal {
public:
    void sound() override {
        cout << "Dog barks" << endl;
    }
};

int main() {
    // Animal a;  //  Error! Cannot instantiate abstract class
    Dog d;
    d.sound();  //  Works fine

    return 0;
}

Base class Animal is abstract and forces derived classes to define sound().

Why Use Pure Virtual Functions?

Encourages consistent interface Ensures all derived classes follow the same structure.
Promotes reusability Common functionality is maintained in the base class.
Supports dynamic polymorphism Allows function calls via base class pointers.

Real-World Applications of Pure Virtual Functions

  1. Shape Hierarchy Shape base class with draw() function for Circle, Square, Triangle.
  2. Payment Gateways PaymentMethod base class with processPayment() for different methods.
  3. File Handling FileFormat base class with openFile() for TextFile, PDFFile, ImageFile.
  4. Device Drivers Device base class with connect() function for USB, Bluetooth, Wi-Fi.

Key Takeaways

A pure virtual function is a function declared in a base class with = 0.
A class with at least one pure virtual function becomes an abstract class.
Derived classes must override pure virtual functions.
Used in real-world applications like payment systems, file handling, and drivers.

10. Destructors in C++

A Destructor is a special member function that is automatically called when an object goes out of scope or is deleted. It is used to free resources like memory, file handles, or network connections when an object is no longer needed.

Definition of Destructor

A member function with the same name as the class but prefixed with ~ (tilde).
It does not take parameters and does not return a value.
Called automatically when an object is destroyed.

Syntax of Destructor

cpp
class ClassName {
public:
    ~ClassName() {
        // Cleanup code
    }
};

The destructor name is always prefixed with ~ and has no parameters.

Real-World Example of a Destructor

When you exit a hotel, the staff cleans the room and releases it for the next guest. Similarly, a destructor cleans up resources when an object is destroyed.

Example: Destructor in Action

cpp
#include <iostream>
using namespace std;

class Car {
public:
    // Constructor
    Car() {
        cout << "Car object created!" << endl;
    }

    // Destructor
    ~Car() {
        cout << "Car object destroyed!" << endl;
    }
};

int main() {
    Car myCar;  // Object is created

    cout << "Inside main function" << endl;

    return 0;  // Object is automatically destroyed here
}

Output:

css
Car object created!
Inside main function
Car object destroyed!

The destructor is automatically called when myCar goes out of scope at the end of main().

Why Are Destructors Needed?

Automatic Cleanup Prevents memory leaks by releasing allocated resources.
File & Database Handling Ensures open files and database connections are closed properly.
Avoids Resource Wastage Deletes dynamically allocated memory and prevents memory leaks.

Example: Destructor Releasing Memory

If an object dynamically allocates memory, we must release it in the destructor.

cpp
#include <iostream>
using namespace std;

class Student {
private:
    string* name;  // Dynamically allocated memory

public:
    // Constructor
    Student(string n) {
        name = new string(n);  // Allocating memory
        cout << "Student object created: " << *name << endl;
    }

    // Destructor
    ~Student() {
        cout << "Releasing memory for: " << *name << endl;
        delete name;  // Freeing memory
    }
};

int main() {
    Student s1("Alice");
    Student s2("Bob");

    return 0;  // Objects go out of scope and destructors are called
}

Output:

yaml
Student object created: Alice
Student object created: Bob
Releasing memory for: Alice
Releasing memory for: Bob

The destructor ensures that dynamically allocated memory is released when objects are destroyed.

Key Features of Destructors

Called automatically when an object is destroyed.
Used to clean up resources like memory, files, and network connections.
Cannot be overloaded (only one destructor per class).
Does not take arguments or return values.
Always declared with ~ (tilde) before the class name.

Real-World Applications of Destructors

  1. File Handling Closes files when an object handling file operations is destroyed.
  2. Database Connections Ensures proper closing of database connections.
  3. Network Communication Releases network sockets and connections after use.
  4. Memory Management Frees dynamically allocated memory to prevent memory leaks.
